# 设计规范文档

## 1. 代码设计原则

### 1.1 分层设计

```
┌─────────────────────────────────────┐
│          Crawler (编排层)            │
│   - 阶段1: 获取商品列表              │
│   - 阶段2: 逐个获取详情并下载        │
│   - 阶段3: 汇总结果                  │
├─────────────────────────────────────┤
│     ProgressManager (进度管理层)     │
│   - progress.json 读写              │
│   - 断点续传追踪                    │
├─────────────────────────────────────┤
│       APIClient (API 客户端层)       │
│   - 请求封装 + 重试 + 延迟          │
│   - UA 轮换 + 代理                  │
├─────────────────────────────────────┤
│     ImageDownloader (下载层)         │
│   - 图片URL提取                     │
│   - 并发下载 + 校验 + 去重          │
└─────────────────────────────────────┘
```

### 1.2 设计原则

- **关注点分离**：每个类只负责一个职责
- **配置外部化**：所有可变参数集中在 config.py
- **防御式编程**：充分的重试、超时、异常处理
- **可观测性**：完整的日志体系，便于排查问题

## 2. 命名规范

### 2.1 Python 代码

| 类别 | 规范 | 示例 |
|------|------|------|
| 类名 | PascalCase | `ProgressManager`, `APIClient` |
| 函数/方法 | snake_case | `fetch_product_list()`, `download_single_image()` |
| 变量 | snake_case | `product_id`, `image_urls` |
| 常量 | UPPER_SNAKE | `MAX_RETRIES`, `REQUEST_DELAY` |
| 文件 | snake_case | `check_source.py`, `crawler.py` |

### 2.2 图片文件命名

```
{名称缩写}_{商品编码}_{序号}.jpg
  ↑            ↑         ↑
 产品名缩写    商品ID    序列号(01,02...)

示例: Cotton_Linen_Shirt_E477181-000_01.jpg
```

- 名称缩写：取 `|` 前的主要名称，去除非字母数字字符
- 名称长度：不超过 40 个字符
- 商品编码：保留原始格式（如 E477181-000）
- 序号：从 01 开始递增

## 3. 错误处理规范

### 3.1 重试策略

| 场景 | 策略 | 参数 |
|------|------|------|
| HTTP 429 (频率限制) | 等待 10×attempt 秒 | 最多3次 |
| HTTP 5xx | 指数退避 1s→2s→4s | 最多3次 |
| 网络超时 | 指数退避 1s→2s→4s | 最多3次 |
| 连接错误 | 固定延迟 1×attempt 秒 | 最多3次 |
| 连续失败 | 累计5次后终止爬虫 | 全局计数器 |

### 3.2 异常处理层级

```
Crawler.run()
  └─ try/except (顶层兜底)
  └─ APIClient._request() (内部重试)
  └─ ImageDownloader.download_single_image() (内部重试)
  └─ ProgressManager.save() (文件IO异常)
```

### 3.3 图片校验

- 最小文件大小：1KB (1024 bytes)
- 格式验证：Pillow `Image.verify()`
- MD5 去重：同一文件不重复下载

## 4. 安全性考虑

- 不存储任何用户凭据
- 不执行任意代码
- 遵守 robots.txt 规则
- 请求间隔控制，避免对目标服务器造成压力
- 通过代理访问（用户自备 VPN）

## 5. 日志规范

### 5.1 日志级别

| 级别 | 用途 | 输出 |
|------|------|------|
| INFO | 正常流程信息 | 控制台 + 文件 |
| WARNING | 可恢复的异常 | 控制台 + 文件 |
| ERROR | 不可恢复的异常 | 控制台 + 文件 + 独立错误日志 |
| DEBUG | 调试细节 | 仅文件日志 |

### 5.2 日志格式

```
2026-06-16 21:38:38 [INFO]   本页 3 个，累计 69/85
2026-06-16 21:39:43 [ERROR]  请求失败(详情 E455957-000): Read timed out
```
