## 3. API 接口规范

### 3.1 优衣库 Commerce API

**基础URL**：`https://www.uniqlo.com/us/api/commerce/v5/en`

#### 商品列表

```
GET /products
  ?path={genderId},{categoryId},,
  &offset={offset}
  &limit={limit}
  &imageRatio=3x4
```

**响应格式**：
```json
{
  "status": "ok",
  "result": {
    "items": [
      {
        "productId": "E477181-000",
        "name": "Cotton Linen Shirt | Short Sleeve",
        "representativeColorDisplayCode": "37",
        "l1Id": 477181,
        "images": {
          "main": { "37": { "image": "https://..." } },
          "sub": [ { "image": "https://...", "colorCode": "64" } ]
        },
        "pagination": { "total": 85, "offset": 0, "count": 36 }
      }
    ]
  }
}
```

#### 商品详情

```
GET /products/{productId}
```

**响应格式**：同列表项中的商品数据结构，更完整

### 3.2 API 参数说明

| 参数 | 类型 | 说明 | 示例 |
|------|------|------|------|
| path | string | 分类路径（逗号分隔ID） | 22211,95669,, |
| offset | int | 分页偏移量 | 0 |
| limit | int | 每页数量（最大36） | 36 |
| imageRatio | string | 图片比例 | 3x4 |
| productId | string | 商品编码 | E477181-000 |

### 3.3 分类 ID 说明

| ID | 含义 |
|----|------|
| 22211 | 男装 (Gender) |
| 95669 | 衬衫/卫衣 (Shirts & Polos) |
| 95668 | 相关子分类 |

## 4. 数据流设计

```
┌─────────────┐     ┌──────────────┐     ┌──────────────┐
│ Listing API  │────▶│  Detail API  │────▶│  Image URLs  │
│ (自动翻页3页) │     │ (逐个，1.5s间隔)│     │ (主图+轮播图)  │
└─────────────┘     └──────────────┘     └──────┬───────┘
                                                ▼
                                        ┌──────────────┐
                                        │  Concurrent   │
                                        │  Downloader   │
                                        │  (8线程)      │
                                        └──────┬───────┘
                                               ▼
                                        ┌──────────────┐
                                        │   images/    │
                                        │  本地存储     │
                                        └──────────────┘
```

## 5. 关键技术决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 数据获取方式 | API 策略 | 发现 Commerce API，返回结构化的JSON数据 |
| 列表页方案 | API offset 分页 | 无需 Selenium，API 直接返回所有商品 |
| 图片范围 | 仅默认颜色 | 减少下载量，满足核心需求 |
| 并发模型 | ThreadPoolExecutor | 图片下载是 IO 密集型任务 |
| 断点续传 | JSON 文件 | 轻量无依赖 |
