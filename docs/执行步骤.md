# 执行步骤文档

**目标**：与用户确认需求，不写任何代码。

**执行记录**：

1. 读取 `E:\workpace\图片爬取.docx` 文档，获取项目信息和页面结构特征
2. 确认项目名称：**uniqlo_spider**
3. 确认图片范围：**仅商品主图区域**
4. 确认探测策略：**先探API再Selenium**
5. 确认高级功能：**开启图片去重（MD5）**
6. 确认图片 XPath：`//img[@class="image__img"]/@src`
7. 打开示例详情页分析实际DOM结构

### 第二阶段：环境搭建与探测

**目标**：搭建环境、编写探测脚本、确认爬取策略。

**执行记录**：

1. 创建项目文件夹 `E:\workpace\uniqlo_spider`
2. 创建 Python 虚拟环境 `venv`
3. 安装依赖：requests, selenium, lxml, Pillow
4. 编写 `check_source.py`（7项测试 + 详情页分析）
5. 用户运行探测脚本，返回结果
6. **分析结果**：
   - ❌ 静态HTML不包含商品链接
   - ❌ XPath在静态HTML中匹配不足（JS动态渲染）
   - 💥 Selenium超时（代理配置问题）
   - ❌ 通用API探测未发现端点
   - ✅ 反爬检测通过
7. 用户手动找到真实 API：
   ```
   https://www.uniqlo.com/us/api/commerce/v5/en/products?path=22211,95669,,&offset=0&limit=36...
   ```
8. 验证API返回JSON数据（85个商品）
9. 更新 `check_source.py` v2，改进API探测逻辑：
   - 从页面HTML提取分类ID
   - 搜索JS bundle中的API构造模式
   - 用提取的ID构建不同path参数组合测试
10. 重新探测成功，发现4个API端点

**产出**：
- `check_source.py`（v2，深度API探测）
- `check_result.json`

**关键发现**：
- Commerce API：`/api/commerce/v5/en/products`
- 分类 ID：22211（男装），95669（衬衫）
- 图片数据结构：`images.main`（主图），`images.sub`（轮播图）

### 第三阶段：正式爬虫

**目标**：编写爬虫代码，运行并下载图片。

**执行记录**：

1. 确认爬取范围：**仅默认颜色的所有轮播图**（不是所有颜色）
2. 命名规则：**名称缩写+商品编码**
3. 编写 `config.py`（所有配置集中管理）
4. 编写 `crawler.py`（4个类：ProgressManager, APIClient, ImageDownloader, Crawler）
5. 语法检查通过
6. 小规模测试（3个商品）：
   - ✅ 商品列表API获取成功
   - ✅ 商品详情API获取成功
   - ✅ 图片下载成功（4张图）
   - ✅ 文件名格式正确
7. 清理测试数据
8. 优化超时和重试参数
9. 用户正式运行爬虫

**产出**：
- `config.py`（配置文件）
- `crawler.py`（正式爬虫）
- `images/`（下载图片目录）
- `logs/`（运行日志）
- `progress.json`（断点续传进度）
